## ЁЯФм Research (Thesis)

### MemoryArchitect: Policy-Driven Memory Governance for LLM Agent Systems
**Arizona State University (ASU) ┬╖ Jul 2025 тАУ Present**

> Most agentic LLM systems fail quietly in long-horizon tasks: either the full chat history is stuffed into the prompt until the context window breaks, or na├пve RAG retrieves *similar* but not *useful* content, silently averaging conflicting facts into hallucinations. The real problem is not "retrieve more." It is *govern what enters memory, how long it survives, what gets retrieved, and how contradictions are resolvedтАФall under hard token budgets, without retraining the model.*

**What I built**

**MemoryArchitect** is a model-agnostic, external memory governance layer for LLM agents that treats memory as a **constrained, auditable resource** rather than a passive log. The same governance engine can harden many different agent systems in real settingsтАФno base-model retraining required.

| Governance Stage | What It Controls |
|---|---|
| **Write policy** | What gets stored тАФ filters noise, duplicates, and injection attempts at ingest time |
| **Metadata & provenance** | Each memory item is tagged with trust score, time scope, sensitivity level, and source |
| **TTL / decay** | When memories expire or fade тАФ configurable forgetting behavior per memory type |
| **Episodic тЖТ semantic consolidation** | How recent traces are compressed into durable, compact summaries |
| **Retrieval eligibility** | What is *allowed* to be retrieved тАФ policy-gated, not just similarity-ranked |
| **Contradiction detection** | Conflicting facts are flagged and resolved before reaching the prompt |
| **Token budget arbitration** | Memory items compete for context slots by benefit-per-token under a hard window constraint |
| **Compliance layer** | Right-to-be-forgotten deletion cascades; toxic/injection-resistant "don't store this" rules |

**Core Technical Ideas**

- **Governance-first memory** тАФ Explicit, configurable policies govern every stage of the memory lifecycle: write, store, decay, consolidate, retrieve, and resolve. No implicit "just embed and retrieve."
- **Contradiction handling** тАФ Conflicting facts are detected via semantic similarity + logical consistency checks and resolved (most recent / highest trust wins) before context assembly, preventing the agent from hallucinating averaged facts.
- **Benefit-per-token budgeting** тАФ At runtime, the system ranks policy-eligible memory items by expected task contribution divided by token cost, packs the context window to the hard limit, compresses when needed, and logs every decision for transparency and debugging. Vector stores (Pinecone ┬╖ LanceDB) back the retrieval layer; n8n orchestrates automated workflow triggers and external system integrations.
- **Provenance & auditability** тАФ Every stored item carries metadata (source, timestamp, trust score, sensitivity). Every memory actionтАФwrite, decay, eviction, retrievalтАФis logged, making agent behavior inspectable and debuggable.
- **Compliance-ready** тАФ Deletion cascades propagate right-to-be-forgotten requests across the memory store. Write-time rules block toxic content and prompt-injection patterns from entering memory at all.

**Initial Evaluation Results**

> Our governance engine outperformed most existing memory baselines across long-horizon recall, persona consistency, and contradiction rate. Crucially, **context window utilization stayed below 47% at all times**тАФeven across arbitrarily long conversationsтАФdemonstrating that tighter memory governance, not larger context windows, is the path to cost- and latency-efficient agentic systems that maintain long-term conversations perfectly.

**Impact**

| Dimension | Outcome |
|---|---|
| ЁЯТ░ Cost & latency | Token usage held **below 47% of the context window at all times**, reducing per-turn cost and inference latency |
| ЁЯза Long-term conversation quality | Episodic consolidation + contradiction resolution enables **perfect long-term conversation maintenance** across extended multi-turn conversations |
| ЁЯОп Persona consistency | Policy-enforced identity traces prevent agent drift over extended, multi-session conversations |
| ЁЯЪл Hallucination reduction | Contradiction resolution stops conflicting facts from reaching the model тАФ no more "averaged" hallucinations |
| ЁЯФТ Deployable compliance | Right-to-be-forgotten + injection-resistant write rules, production-ready out of the box |
| ЁЯФМ Model-agnostic | Governance layer is fully decoupled from the base model тАФ the same engine hardens any LLM agent stack |

**AGI relevance** тАФ Scalable general-purpose intelligence requires memory that is controlled, auditable, and useful. Agents need governance policiesтАФnot retrieval heuristicsтАФfor reliable recall. Resource allocation under hard constraints is a foundational problem for any long-horizon reasoning system.
